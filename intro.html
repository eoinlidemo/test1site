Welcome to the introduction of this demo.

上下滚动网页，你会发现本站的网页均由 3 个主要部分组成。

  • 顶部的导航栏（页眉），滚动网页时不移动；
  • 中部的主要内容，其背景图滚动网页时不移动；
  • 底部的页脚，包含与本项目有关的链接。

这 3 个部分，分别使用 3 个元素 header、section 和 footer 编写。
HTML 网页中的所有可显示出的内容，全部都是元素。
如你所见，这里写的 4 行文字，都在一个 p 元素的两个标签之内。
元素分为单标记元素和双标记元素，它们的区别在于……

<p>这是一个双标记元素，首尾分别有一个标记，各表示元素的开始和结束。</p>
<img 这是一个单标记元素，有关设置全部写在一个标记内部的斜杠之前。/>

源代码中，我们用一个 class 为 navbar 的 div 元素来制作导航栏。
div 元素是一个没有实际意义的块元素。块元素独占一行，我们可以给它设置上下左右任意方向的内外边距。
导航栏中的每一个部分，包括左边的 DEMO 大字和右边的菜单项，它们的 class 属性均包含 li 这个名字。
上面所说的 class 和未提及的 id，均为 HTML 元素的属性。它们的存在让开发者得以对特定的元素设置显示样式。
这两者的区别在于，id 具有唯一性，而 class 则不具有。全局上说，id 只能出现在一个特定的元素身上。
开发者在 CSS 文件中通过形如 .class、#id 的方式指定特定元素并为其设定样式，. 和 # 符号因而分别被称为选择器。

.类名{样式}
#id名{样式}

除此之外，直接使用元素名也可以设置样式，被称为元素选择器。这一点使用得也很常见：
如果不加设置，根据 Google Chrome 等浏览器的预设，浏览器会自动给网页正文加入 8 像素的外边距，很影响美观。
因此大部分网页在设置样式时，都会给正文部分设置外边距 margin 为 0 来覆盖掉浏览器的预设。
实现这一点的 CSS 样式代码非常简单—— body{margin: 0;} 如你所见，在此网站的网页中，这一点也得到了体现。

接下来就是一个很复杂的部分——定位。
每个元素在网页中都会有特定的位置来显示，比如这个网页中的每一个写有文字的黑色方块都分别设定了各自的位置。
设定位置这项工作也需要开发者在 CSS 样式代码中完成，需要用到元素的定位属性，position。
这个属性可以填入 5 个值：static、relative、absolute、fixed 以及 sticky。
当然，position 属性的默认值为 static。这意味着，如果不设置属性值，则定位默认为 static。
为了更加直观地展现这些值的效果，这里引用了 MDN Web Docs 所做的交互式控件，以供参考。
回链：https://interactive-examples.mdn.mozilla.net/pages/css/position.html

static 作为默认值，它的效果最为普通：样式设置成咋样就咋样；
relative 字面意思是 “相对的”，它表示在元素原本的位置（即设置为 static 时的位置）上加以调整；
absolute 字面意思是 “绝对的”，它表示在相对于元素最近的父级元素设定位置；
fixed 字面意思是 “固定的”，它表示元素相对于浏览器页面的位置不发生改变；
sticky 字面意思是 “粘性的”，元素可完整显示在浏览器页面中的时候，该元素定位为 relative；当元素不能完整显示的时候（比如滚动到显示范围之外的时候），该元素定位为 fixed。这么说或许有点不容易理解，因此我做了一个控件来演示。
链接：https://codepen.io/eoinlidemo/pen/qBXOwyB/

上面两个演示中，或许你已经注意到，还设置了 top 和 left 两个属性，他们分别代表元素顶部和左侧偏移的大小。
值得注意的是，这两个属性不会对 position 设置为 static 的元素生效。

这个项目中，顶部的导航栏采用 fixed 定位，背景采用 absolute 定位，文字框采用 relative 定位。
每一个文字框都通过 JavaScript 分别设置了 top 属性的值，且分别递增 30 像素，这样能够保证相邻两个文字框之间的垂直间隔均为 30 像素。
其逻辑在于，如果给文字框添加 static 定位，则所有的文字框将依次往下排放，中间没有空隙。第一个的文字框距离顶部有 120 像素高，为了保证文字框间距 30 像素，则需使用 relative 定位，将之后每一个文字框的 top 属性都递增 30 像素。
视频后半部分的红色文字框表示文字框原本的位置，绿色框表示文字框设置样式后的位置。

在网页设计中，HTML 和 CSS 被称为前端，因为这两个语言通常用来设计网页可以显示出的内容，也就是呈现给用户的内容；
而刚才所说的 JavaScript，简称 JS，被称为后端，因为我们一般用这个语言来编写网页背后的计算和信息处理的任务，和 Python 有些许相似。
为了避免把 CSS 写得过于重复和复杂，批量给元素添加相同的样式的时候，JavaScript 承担重任。
本网站上网页的所有文字框的定位，都是使用 JavaScript 设置的。

var tb = document.getElementsByClassName("box1");

上面这行 JS 代码，获取了网页内每一个 class 为 box1 的元素，并将它们全部编进了一个列表——或者在 JS 中，我们称之为数组。这一点 JS 和 Python 非常相似。
在 Python 中，for 循环用来遍历列表中的每一个元素，对遍历到的元素执行相同的代码。
JS 同样也有 for 循环，只不过结构有些许不同——

for ([initialization]; [condition]; [final-expression]){statement}

[initialization] 指用来表示每一个被遍历元素的变量，一般设为 var i=0；
[condition] 指用来判断循环是否继续进行下去的判断句；
[final-expression] 指每一次循环结束后执行的代码，一般设为 i++；
{statement} 指循环的执行语句。

这样，在 i 为 0 的时候，不会执行语句，因为第一个文本框的 top 属性已经于 CSS 设置好了；
在 i 为 1 的时候，第二个文本框的 top 属性为 120 + 30 * 1 = 150 px;
在 i 为 2 的时候，第二个文本框的 top 属性为 120 + 30 * 2 = 180 px;
以此类推。
